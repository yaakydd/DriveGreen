1. What is Tailwind CSS?
Tailwind is a utility-first CSS framework that gives you low-level utility classes to build custom designs without leaving your HTML.

Key Benefits:

No custom CSS needed

Consistent design system

Faster development

Responsive by default

Highly customizable

2. Setup & Installation
For React/Vite Project:
bash
# Create new project
npm create vite@latest my-app -- --template react

# Install Tailwind
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
Configure tailwind.config.js:
javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      // Add custom colors, fonts, etc.
      colors: {
        'custom-green': '#10b981',
        'custom-blue': '#3b82f6',
      },
      fontFamily: {
        'mono': ['JetBrains Mono', 'monospace'],
      },
    },
  },
  plugins: [],
}
Add Tailwind to CSS (index.css or App.css):
css
@tailwind base;
@tailwind components;
@tailwind utilities;

10. Responsive Design
Breakpoints
Tailwind uses a mobile-first breakpoint system:

jsx
// Mobile first (default)
<div className="text-sm">Small on mobile</div>

// Responsive modifiers
<div className="text-sm md:text-base lg:text-lg xl:text-xl">
  Responsive text
</div>

<div className="w-full md:w-1/2 lg:w-1/3 xl:w-1/4">
  Responsive width
</div>

<div className="block md:flex">
  Stack on mobile, flex on desktop
</div>
Breakpoint Ranges:
Prefix	Min-width	CSS
sm:	640px	@media (min-width: 640px)
md:	768px	@media (min-width: 768px)
lg:	1024px	@media (min-width: 1024px)
xl:	1280px	@media (min-width: 1280px)
2xl:	1536px	@media (min-width: 1536px)
Responsive Patterns
jsx
// Hide/show elements
<div className="hidden md:block">Visible on medium+</div>
<div className="block md:hidden">Visible only on mobile</div>

// Responsive grid
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4">

// Responsive spacing
<div className="p-4 md:p-6 lg:p-8">

// Responsive typography

13. Best Practices
1. Component Composition
jsx
// ‚ùå Don't repeat long class strings
<div className="bg-white p-6 rounded-lg shadow-lg border border-gray-200 ...">

// ‚úÖ Extract to components
const Card = ({ children, className = '' }) => (
  <div className={`bg-white p-6 rounded-lg shadow-lg ${className}`}>
    {children}
  </div>
);

// Usage
<Card className="border border-gray-200">
  Content
</Card>
2. Use @apply for Reused Styles
css
/* In your CSS file */
.btn-primary {
  @apply px-6 py-3 bg-emerald-600 text-white rounded-lg 
         hover:bg-emerald-700 transition-colors duration-200;
}

.btn-secondary {
  @apply px-6 py-3 bg-gray-100 text-gray-800 rounded-lg 
         hover:bg-gray-200 transition-colors duration-200;
}
3. Responsive Design Order
jsx
// Correct order: mobile first, then responsive
<div className="text-sm md:text-base lg:text-lg">‚úÖ

// Wrong order
<div className="lg:text-lg md:text-base text-sm">‚ùå
4. Keep Class Names Readable
jsx
// Use line breaks for complex classes
<button className="
  px-6 py-3 
  bg-gradient-to-r from-emerald-500 to-cyan-500 
  text-white font-semibold 
  rounded-lg shadow-lg 
  hover:shadow-xl hover:scale-105 
  transition-all duration-300 
  focus:outline-none focus:ring-2 focus:ring-emerald-500
">
  Button
</button>
5. Use className Libraries for Dynamic Classes
jsx
import clsx from 'clsx'; // or classnames

const Button = ({ variant, size, disabled }) => (
  <button className={clsx(
    'px-4 py-2 rounded-lg transition-colors',
    {
      'bg-emerald-600 text-white': variant === 'primary',
      'bg-gray-200 text-gray-800': variant === 'secondary',
      'px-6 py-3 text-lg': size === 'large',
      'opacity-50 cursor-not-allowed': disabled,
    }
  )}>
    Button
  </button>
);
